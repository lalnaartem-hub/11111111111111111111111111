# Browser OS

Операционная система в браузере (TypeScript + React + Vite). Виртуальная файловая система, окна, терминал, настройки и эмулятор x86 (FreeDOS + v86).

## Запуск (Windows)

1. Установите [Node.js LTS](https://nodejs.org/) (или `winget install OpenJS.NodeJS.LTS`).
2. Откройте **новый** терминал в папке проекта.
3. Выполните:

```powershell
cd C:\Users\lalna\Desktop\1111111
npm install
npm run setup:v86
npm run dev
```

Либо дважды щёлкните `start-dev.cmd` (добавляет Node в PATH и запускает dev-сервер).

Откройте http://localhost:5173/

## Запуск .exe / .com

| Тип файла | Результат |
|-----------|-----------|
| DOS `.COM`, DOS `.EXE` (MZ, не PE) | Эмулятор + FreeDOS + дискета A: |
| Windows PE `.exe` | Не поддерживается (нужна полная Windows) |

1. **Файлы** → `/home/HELLO.COM` → «Запустить» или двойной клик.
2. **Эмулятор** → «Открыть .exe / .com» → «Запустить с .exe».

В DOS после загрузки: `A:` → имя файла (8.3, например `HELLO.COM`).

## Настройки

- Обои (градиенты, без внешних URL — совместимо с эмулятором)
- Dock: позиция, увеличение иконок
- Окна: прозрачность, размытие, тени, акцент
- Эмулятор: объём RAM (32–128 МБ)
- Система: анимации, восстановление сессии, журнал, горячие клавиши

## Горячие клавиши

- `Ctrl+F` — Файлы
- `Ctrl+T` — Терминал
- `Ctrl+,` — Настройки
- `Ctrl+E` — Редактор
- `Ctrl+Shift+E` — Эмулятор
- `Ctrl+W` — закрыть активное окно
- `Ctrl+M` — свернуть

## Скрипты

| Команда | Описание |
|---------|----------|
| `npm run dev` | Dev-сервер |
| `npm run build` | Сборка |
| `npm run type-check` | Проверка TypeScript |
| `npm run setup:v86` | Скачать WASM и BIOS для эмулятора |

## Структура

- `src/` — приложение, bootstrap, настройки
- `packages/` — ядро, ФС, окна, GUI, эмулятор
- `.kiro/specs/browser-os/` — требования и план
- `public/v86/`, `public/bios/` — файлы эмулятора (после setup)

## Ограничения

Полный план в `tasks.md` (тесты, PE-Windows, workers, App Store с установкой) ещё не завершён. Текущая сборка ориентирована на рабочий UI, ФС, сессии и DOS-эмуляцию.
